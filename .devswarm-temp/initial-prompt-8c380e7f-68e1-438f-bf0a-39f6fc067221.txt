## What is DevSwarm?

DevSwarm is a **multi-tasking IDE** that lets developers work continuously without waiting on AI. Instead of one AI assistant working on one thing at a time, DevSwarm lets you:

- **Work on multiple features simultaneously** — each in its own isolated workspace
- **Never wait for AI** — while one workspace handles a task, start another
- **Run your app in parallel** — port variables let you run your full application stack in every workspace simultaneously
- **Maintain clean git history** — each workspace is a separate git worktree on its own branch

Each workspace is a **complete IDE** with its own AI assistant, file explorer, terminal, and independent state. You can have 3 workspaces open, each running the full app, each with AI working on different features — and switch between them instantly.

## DevSwarm CLI

The `devswarm` CLI is available in your terminal:

```bash
devswarm workspace create <branch> -p "<prompt>" -t "<title>"  # Create child workspace
devswarm workspace list                                        # List workspaces
devswarm workspace info                                        # Get workspace details
devswarm workspace message-parent "<msg>"                      # Message parent
devswarm workspace message-child <branch> "<msg>"              # Message child
devswarm workspace read-messages                               # Read messages
devswarm workspace monitor                                     # Wait for messages
devswarm workspace merge-from-source                           # Pull from source
devswarm workspace merge-into-source                           # Merge into source
devswarm workspace check-merge                                 # Check for conflicts
```

## When to suggest parallel workspaces

**Good candidates for parallel work:**
- Independent features (dark mode + user settings + notifications)
- Bug fixes that don't overlap with feature work
- Refactoring one area while adding features to another
- Multiple small tasks that together would take all day

**Keep on same workspace:**
- Tightly coupled changes that depend on each other
- Quick fixes (< 5 minutes)
- Exploratory/research work
